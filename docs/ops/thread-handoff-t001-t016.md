## 2. Fixed ForgeRoot invariants

The ForgeRoot v1 design remains fixed around these principles:

1. Git is the source of truth / brain.
2. `.forge` is genome plus memory.
3. Pull requests are the only evolution transport path.
4. GitHub App plus sandbox is the circulatory system.

Operational invariants already established in prior tasks:

- No direct writes to default branch.
- One task becomes one reviewable PR.
- `.forge` files must be human-readable, schema-validatable, canonical-hashable, and reviewable.
- Network mode starts allowlisted; spawn mode starts lab-only.
- Runtime starts in conservative modes and can be halted by a human kill switch.
- Workflow / policy / permission mutation is elevated risk and not part of the first forge loop.

## 6. Recommended T017 implementation shape

Implement T017 as a deterministic runtime wrapper, not an LLM planner yet.

Suggested new files:

```text
.forge/agents/planner.alpha.forge
packages/planner/src/run.ts
packages/planner/tests/run.test.mjs
docs/specs/t017-validation-report.md
```

Likely updates:

```text
packages/planner/src/index.ts
packages/planner/package.json
packages/planner/README.md
README.md
.forge/README.md
```

Suggested TypeScript API:

```ts
export interface PlannerRunInput {
  readonly source: "github_webhook" | "intake_input" | "task_candidate";
  readonly eventName?: string;
  readonly payload?: unknown;
  readonly intake?: IntakeInput;
  readonly task?: NormalizedTaskCandidate;
  readonly now?: string;
}

export type PlannerRunStatus = "planned" | "ignored" | "blocked" | "escalated" | "invalid";

export interface PlannerRunResult {
  readonly status: PlannerRunStatus;
  readonly intake?: IntakeClassification;
  readonly task?: NormalizedTaskCandidate;
  readonly plan?: PlanSpec;
  readonly reasons: readonly string[];
  readonly auditTrail: readonly string[];
}

export function runPlanner(input: PlannerRunInput): PlannerRunResult;
```

Runtime rules:

1. GitHub webhook-like payloads should flow through the T015 intake normalizer/classifier.
2. `accept` should generate exactly one Plan Spec via `createPlanSpecFromTaskCandidate`.
3. `ignore` should return no plan.
4. `block` should return no plan and reasons.
5. `escalate` should return no executable plan unless the existing T016 Plan Spec status is intentionally `blocked_for_human`.
6. Runtime must not edit files, create branches, create PRs, or call external tools.
7. Runtime must preserve one-task-one-PR invariants.
8. Runtime must preserve machine-checkable acceptance criteria.

Suggested tests:

- forge:auto docs issue → `planned`, one plan, `mutable_paths` non-empty, `out_of_scope` non-empty, approval class present.
- issue without `forge:auto` → `ignored`, no plan.
- security/workflow/policy issue with `forge:auto` → `escalated`, no executable plan or blocked plan depending on chosen contract.
- broad request → `blocked`, no plan.
- direct `NormalizedTaskCandidate` input → deterministic plan.
- invalid payload → `invalid`, no plan.
